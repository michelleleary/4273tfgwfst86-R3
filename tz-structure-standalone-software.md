# Структура ТЗ для профиля standalone_software

Если СТРУКТУРА_ИНЖЕНЕРА или ШАБЛОН_ЗАКАЗЧИКА задают иной порядок — использовать их.

## Нормативная основа

- Для программного изделия без контура ОКР использовать ЕСПД или шаблон заказчика.
- ГОСТ 19.201 применять как ориентир для состава ТЗ на программу, если заказчик требует ЕСПД.
- ГОСТ 34.602-2020 применять только если объект является автоматизированной системой, а не отдельной программой.
- Оформление: ГОСТ 2.105-2019 или шаблон заказчика.
- Актуальность стандартов проверять через `search`.

Профиль применим для самостоятельного ПО: веб-сервис, desktop/mobile-приложение, библиотека, модуль, модель, интеграционный сервис, без аппаратного изделия и без обязательной структуры ОКР 1–14.

## Титульный лист

- «ТЕХНИЧЕСКОЕ ЗАДАНИЕ».
- Наименование программного изделия.
- Заказчик и исполнитель, если заданы.
- Версия / шифр / обозначение: пусто, если не задано.
- Год.
