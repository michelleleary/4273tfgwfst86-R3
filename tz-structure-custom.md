# Структура ТЗ для профиля custom

Профиль `custom` используется, когда пользователь, инженер или заказчик передал собственную структуру документа.

## Главный принцип

Сначала применять:

1. `СТРУКТУРА_ИНЖЕНЕРА`;
2. `ШАБЛОН_ЗАКАЗЧИКА`;
3. явные настройки пользователя;
4. только затем типовые профили.

Нельзя заменять пользовательскую структуру типовой схемой, если она не противоречит обязательной нормативной основе.

## Нормативная основа

- Определить нормативный контур по содержанию: ОКР, изделие, ПО, АС, аппаратное изделие, смешанный продукт.
- Проверить актуальность стандартов через `search`.
- Если стандарт требует обязательный раздел или сведения, но их нет в структуре заказчика, добавить их как подпункт или открытый вопрос.
- Если структура заказчика конфликтует со стандартом, не исправлять молча: указать риск в служебном отчёте или задать вопрос.

## Алгоритм сборки структуры

1. Сохранить порядок разделов, переданный пользователем.
2. Нормализовать названия разделов без изменения смысла.
3. Разнести смешанные требования по логическим подразделам.
4. Не добавлять разделы «для красоты».
5. Для отсутствующих обязательных сведений использовать `[УТОЧНИТЬ]`.
6. Не выдумывать шифр, основание, сроки, заказчика, исполнителя, состав, этапы, испытания.

## Карта соответствия

Если пользовательская структура неполная, использовать ближайший профиль как источник недостающих ориентиров:

| Признак | Ближайший профиль |
|---------|-------------------|
| ОКР аппаратного изделия | `okr_product` |
| ОКР программного изделия | `okr_software` |
| ОКР аппаратно-программного комплекса | `okr_mixed_product` |
| Самостоятельное ПО | `standalone_software` |
| Самостоятельное аппаратное изделие | `standalone_hardware` |
| Автоматизированная система | `as_system` |

## Обязательные элементы контроля

- Титульный лист — по шаблону заказчика или общему шаблону ТЗ.
- Перечень сокращений — если есть сокращения.
- Основание разработки — `[УТОЧНИТЬ]`, если не задано.
- Требования — только по входным данным.
- Этапы и приёмка — по договору, структуре инженера или `[УТОЧНИТЬ]`.
- Документация — по профилю и входным данным.

## Что запрещено

- Перестраивать документ под `okr_product`, если выбран `custom`.
- Подставлять несуществующие ГОСТы или статусы стандартов.
- Удалять разделы заказчика без объяснения.
- Делать обязательными требования, которые пользователь не задавал и стандарт прямо не требует.
- Смешивать служебный отчёт генератора с телом ТЗ.
